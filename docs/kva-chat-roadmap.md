# KVA Chat Roadmap

Локальная продуктовая карта под новый messenger-формат.

## Фаза 1. Базовый messenger

- Личные чаты `1-1`
- Мини-группы до `5-10` человек
- Текстовые сообщения
- Ответ на сообщение
- Аватарка профиля и аватарка чата
- Единое отображаемое имя: `имя/фамилия` или `ник`
- Отметка прочтения
- Упоминания через `@`

## Фаза 2. Медиа и файлы

- Голосовые сообщения
- Передача файлов без сжатия
- Передача файлов с быстрым сжатием для скриншотов и фото
- Группировка вложений по аналогии с Telegram
- Поддержка `PNG`, `JPEG`, `PDF`, `DOC/DOCX`, `XLS/XLSX`, `CDR`, `AI`, `PSD` и похожих форматов
- Управление лимитами и хранением:
  - мелкий клиент: `1-5 ГБ / полгода`
  - крупный клиент: `30-40 ГБ / год`
  - разовая отправка: `10-100 МБ`

## Фаза 3. Уведомления и вход

- Push-уведомления на телефон через браузер
- Простое звуковое уведомление
- Упрощенный вход по ссылке
- Сценарий нового клиента:
  - имя
  - должность / компания
  - опционально почта
  - телефон обсуждаем отдельно

## Фаза 4. Внешняя точка входа

- Ссылка с сайта на личный чат
- Новый клиент может сам написать в личные сообщения без долгой регистрации

## Что уже частично есть в текущем проекте

- Текстовые сообщения
- Голосовые каналы
- Личные звонки
- Шеринг экрана в личном звонке
- Передача файлов
- Аватарка профиля

## Что рекомендую брать первым

1. Полноценные личные чаты `1-1`
2. Мини-группы
3. Ответы на сообщения и read receipts
4. Голосовые сообщения
5. Нормальная модель хранения файлов
6. Push-уведомления

## Архитектурный комментарий

Текущая система выросла из group/voice workspace. Для нового продукта правильнее постепенно смещать центр интерфейса к сущности `чат/диалог`, а не `группа/канал`.

То есть следующий большой этап после локального редизайна:

- ввод сущности `conversation`
- типы:
  - `direct`
  - `group`
  - `support-entry`
- единая лента сообщений
- отдельные права доступа и вложения на уровне диалога
